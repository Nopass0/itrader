# Добавление поддержки SQLite в проект iTrader

**Дата**: 2025-05-17
**Автор**: Claude

## Описание

Реализована альтернативная поддержка SQLite в проекте iTrader, что позволяет запускать проект без необходимости установки и настройки PostgreSQL. Это особенно полезно для разработки, тестирования или в ситуациях, когда PostgreSQL недоступен или его установка вызывает сложности.

## Обоснование

Добавление поддержки SQLite обеспечивает следующие преимущества:

1. **Упрощение установки** - SQLite не требует отдельного сервера БД, упрощая настройку проекта
2. **Снижение системных требований** - SQLite работает в режиме файла, не требуя дополнительных системных ресурсов
3. **Портативность** - Вся база данных хранится в одном файле, что упрощает резервное копирование и перенос
4. **Отказоустойчивость** - Если сервис PostgreSQL недоступен, проект все равно может работать на SQLite
5. **Упрощение разработки** - Разработчики могут быстро запустить проект без настройки PostgreSQL

### Использованные технологии

1. **SQLite3**
   - Легковесная встраиваемая реляционная база данных
   - Не требует настройки сервера, работает с файлом базы данных

2. **Prisma ORM с SQLite**
   - Настройка Prisma для работы с SQLite вместо PostgreSQL
   - Модификация схемы для совместимости с SQLite

3. **Node.js скрипты**
   - Реализация скрипта для переключения между базами данных
   - Автоматическое обновление переменных окружения

### Архитектурные решения

1. **Динамическое переключение БД**
   - Возможность выбора между PostgreSQL и SQLite при установке
   - Скрипт для переключения на SQLite в уже настроенном проекте

2. **Сохранение структуры данных**
   - Сохранение схемы данных Prisma для совместимости с обеими БД
   - Адаптация миграций для SQLite

3. **Интеграция в процесс разработки**
   - Добавление специальных npm-скриптов для запуска с SQLite
   - Простой CLI-параметр `--sqlite3` для выбора SQLite

## Примеры использования

### Установка с SQLite

```bash
# Установка с использованием SQLite вместо PostgreSQL
./scripts/db-setup.sh --sqlite3

# Или с использованием npm-скрипта
npm run install:sqlite
```

### Переключение на SQLite в существующем проекте

```bash
# Переключение проекта на SQLite
npm run use:sqlite
```

### Запуск проекта с SQLite

```bash
# Разработка с SQLite
npm run dev:sqlite

# Продакшен с SQLite
npm run start:sqlite
```

## Ограничения

Важно отметить следующие ограничения при использовании SQLite:

1. **Одновременный доступ** - SQLite имеет ограничения при параллельном доступе множества клиентов
2. **Масштабируемость** - Для больших нагрузок или многопользовательской работы рекомендуется PostgreSQL
3. **Функциональность** - Некоторые специфические функции PostgreSQL могут отсутствовать в SQLite
4. **Синхронизация данных** - При переключении между базами данные не синхронизируются

## Рекомендации по использованию

- **Разработка**: SQLite идеально подходит для локальной разработки и тестирования
- **Производственные среды с низкой нагрузкой**: SQLite может использоваться для небольших инсталляций
- **Производственные среды с высокой нагрузкой**: Рекомендуется использовать PostgreSQL
- **Переключение**: При переключении с одной БД на другую данные не переносятся автоматически

SQLite следует рассматривать как альтернативный вариант для ситуаций, когда PostgreSQL недоступен, а не как полноценную замену в продакшене.
