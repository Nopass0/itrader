# Установка PostgreSQL без использования репозиториев Ubuntu

**Дата**: 2025-05-17
**Автор**: Claude

## Описание

Создан скрипт для автоматической установки и настройки PostgreSQL для проекта iTrader, который устанавливает PostgreSQL напрямую с официального сайта, минуя репозитории Ubuntu. Скрипт также настраивает базу данных, создает необходимые учетные записи и устанавливает зависимости проекта.

## Обоснование

Установка PostgreSQL через стандартные репозитории Ubuntu не всегда возможна или желательна:
1. В некоторых средах могут быть ограничения на использование системных пакетных менеджеров
2. Могут возникать проблемы с доступностью серверов репозиториев
3. Может потребоваться конкретная версия PostgreSQL, не доступная в стандартных репозиториях
4. Использование официального установщика гарантирует получение последней стабильной версии

Данный скрипт решает эти проблемы, скачивая официальный установщик напрямую с сайта PostgreSQL и выполняя автоматическую установку.

## Технические детали

### Затронутые файлы

- `/home/user/projects/aitrader/scripts/db-setup.sh` - скрипт установки PostgreSQL
- `/home/user/projects/aitrader/README.md` - обновлены инструкции по установке
- `/home/user/projects/aitrader/AGENTS.md` - обновлен раздел по настройке окружения
- `/home/user/projects/aitrader/docs/agents/2025-05-17_postgresql-setup-script.md` - документация о скрипте

### Использованные технологии

1. **Официальный установщик PostgreSQL**
   - Загрузка установщика напрямую с сайта PostgreSQL
   - Определение архитектуры системы (x86_64 или aarch64)
   - Установка в неинтерактивном режиме с заданными параметрами

2. **Bash скриптинг**
   - Проверка наличия и версии PostgreSQL
   - Обработка разных сценариев установки
   - Динамическое определение пользователя PostgreSQL (postgres или enterprisedb)

3. **Настройка окружения**
   - Добавление установленного PostgreSQL в PATH
   - Настройка пользователей и баз данных
   - Создание конфигурационных файлов проекта

### Архитектурные решения

1. **Универсальность**
   - Скрипт работает как при первой установке, так и при повторных запусках
   - Адаптируется к разным типам установки PostgreSQL (из репозитория или из официального установщика)

2. **Безопасность**
   - Использование временных директорий для загрузки файлов
   - Установка паролей для пользователей баз данных

3. **Обработка ошибок**
   - Проверка результатов каждого этапа установки
   - Информативные сообщения об ошибках

## Тестирование

Проведены следующие тесты:

1. **Установка на чистой системе**
   - Успешная установка PostgreSQL на системе без ранее установленной СУБД
   - Корректное создание пользователя и базы данных

2. **Повторный запуск**
   - Корректная обработка ситуации, когда PostgreSQL уже установлен
   - Отсутствие ошибок при повторном создании пользователя и базы данных

3. **Проверка работоспособности**
   - Успешное соединение с базой данных после установки
   - Применение миграций Prisma
   - Создание тестового администратора

## Дополнительные рекомендации

1. **Запуск скрипта**
   - Рекомендуется запускать с правами администратора: `sudo ./scripts/db-setup.sh`
   - При запуске без sudo может потребоваться ввод пароля в процессе установки

2. **Настройка после установки**
   - Проверить, что сервис PostgreSQL запущен: `sudo systemctl status postgresql-15.4`
   - Убедиться, что PostgreSQL добавлен в PATH

3. **Возможные проблемы**
   - Если установка через официальный установщик не удалась, можно попробовать альтернативные методы установки, например, через Docker
