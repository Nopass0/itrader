## Возможности

- **Интеграция с биржами**:
  - Gate.cx API - транзакции, SMS и push-уведомления
  - Bybit API - ордера, депозиты, выводы

- **Пользовательский интерфейс**:
  - Современный дизайн с глазеоморфизмом
  - Адаптивная верстка для всех устройств
  - Темная и светлая темы

- **Безопасность**:
  - JWT аутентификация
  - Управление пользователями и правами доступа
  - Безопасное хранение API-ключей

- **Управление через CLI**:
  - Интерактивный консольный интерфейс
  - Управление пользователями и сессиями
  - Проверка базы данных и мониторинг работы

## Установка

### Для новой среды разработки (Ubuntu)

#### Настройка базы данных и зависимостей

Используйте скрипт настройки базы данных:

```bash
# Для PostgreSQL (по умолчанию)
./scripts/db-setup.sh

# Для SQLite
./scripts/db-setup.sh --sqlite3
# или
npm run install:sqlite
```

Этот скрипт:
- Устанавливает PostgreSQL напрямую с официального сайта (если не установлен и не выбран SQLite)
- Настраивает PostgreSQL или SQLite (в зависимости от выбора)
- Создает конфигурационные файлы для сервера и фронтенда
- Устанавливает зависимости проекта
- Применяет миграции Prisma
- Создает учетную запись администратора по умолчанию
- Проверяет соединение с базой данных

**Примечание**: Для запуска скрипта с PostgreSQL требуются права администратора (sudo).
**Важно**: PostgreSQL устанавливается напрямую с официального сайта, без использования репозиториев Ubuntu.

#### Использование SQLite вместо PostgreSQL

Если у вас возникли проблемы с установкой PostgreSQL или вы предпочитаете SQLite для разработки, вы можете использовать следующие команды:

```bash
# Установка с SQLite
npm run install:sqlite

# Переключение на SQLite (если уже установлен проект)
npm run use:sqlite

# Запуск разработки с SQLite
npm run dev:sqlite

# Запуск продакшена с SQLite
npm run start:sqlite
```

**Важно**: SQLite рекомендуется использовать только для разработки, тестирования или в ситуациях, когда PostgreSQL недоступен. В продакшене рекомендуется использовать PostgreSQL для лучшей производительности и надежности.

#### Быстрая установка для разработки (one-liner)

Для полностью автоматической установки без запросов на подтверждение используйте:

```bash
curl -s https://raw.githubusercontent.com/username/aitrader/main/scripts/quicksetup.sh | bash
```

Или если вы уже скачали репозиторий:

```bash
./scripts/quicksetup.sh
```

#### Стандартная установка

Используйте скрипт автоматической настройки, который установит все необходимые зависимости, включая Node.js, Bun, PostgreSQL, настроит базу данных и создаст первоначального пользователя:

```bash
# Перейдите в корневую директорию проекта
cd /path/to/aitrader

# Запустите скрипт установки
./scripts/setup.sh
```

Скрипты установки:
- Установят Node.js, Bun, PostgreSQL и другие системные зависимости
- Настроят базу данных PostgreSQL
- Создадут необходимые .env файлы с переменными окружения
- Установят зависимости для всех частей проекта
- Инициализируют базу данных и применят миграции
- Создадут тестового администратора
- Сгенерируют SSL-сертификаты для локальной разработки

### С помощью скрипта интерактивной установки

```bash
# Запустить скрипт установки в интерактивном режиме
./run.sh

# Или установить все зависимости напрямую
./run.sh --install
```

### Ручная установка

```bash
# Установить зависимости сервера
cd server
bun install

# Установить зависимости фронтенда
cd ../frontend
npm install
```

## Запуск


## Разработка

Проект разделен на два основных компонента:

1. **Сервер** - Bun, TypeScript, Elysia.js, Socket.io, Prisma ORM
2. **Фронтенд** - Next.js, TypeScript, Shadcn/UI, Tailwind CSS, Framer Motion

### Структура проекта

```
aitrader/
├── server/              # Серверная часть
│   ├── prisma/          # ORM и миграции БД
│   └── src/             # Исходный код сервера
│       ├── config/      # Конфигурация
│       ├── database/    # Управление БД
│       ├── middleware/  # Middleware
│       ├── routes/      # API маршруты
│       ├── services/    # Сервисы
│       └── utils/       # Утилиты
├── frontend/            # Клиентская часть
│   ├── app/             # Страницы Next.js
│   ├── components/      # UI компоненты
│   ├── hooks/           # React хуки
│   ├── lib/             # Утилиты
│   ├── store/           # State management
│   └── types/           # TypeScript типы
├── docs/                # Документация
├── scripts/             # Скрипты для установки и запуска
└── docker-compose.yml   # Docker Compose конфигурация
```

## Деплой

### С использованием Docker

1. Убедитесь, что Docker и Docker Compose установлены
2. Запустите `npm run docker:build` для сборки образов
3. Запустите `npm run docker:up` для запуска контейнеров
4. Приложение будет доступно по адресу:
   - Frontend: http://localhost:3001
   - API: http://localhost:3000

### Ручной деплой

1. Соберите приложение: `npm run build`
2. Запустите приложение: `npm run start`
